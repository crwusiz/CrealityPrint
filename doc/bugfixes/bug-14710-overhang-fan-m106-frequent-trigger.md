# Bug 修复说明

## 1. 基本信息
- Bug ID: `14710`
- 标题: `短距离悬垂路径切换导致 M106 频繁触发`
- 日期: `2026-02-24`
- 反馈人:
- 处理人: `wangwenbin`
- 分支/提交:

## 2. 问题现象
- 在包含大量短距离悬垂的模型中，切片后 G-code 会在相邻短路径切换时频繁插入 `M106`。
- 常见表现为悬垂风扇在极短时间内反复开启/恢复（例如 `3127 -> 3229 -> 3127 -> 3229`）。
- 影响:
  - 风扇频繁切换没有实际冷却收益（风扇存在响应滞后）。
  - 打印过程产生无意义控制指令，影响稳定性与可读性。
  - 局部悬垂质量可能反而变差（风量无法稳定建立）。

## 3. 影响范围
- 模块: `G-code CoolingBuffer 风扇控制逻辑`
- 关键文件:
  - `src/libslic3r/GCode/CoolingBuffer.cpp`
- 受影响流程:
  - 启用悬垂/桥接风扇标记的 FDM 切片 G-code 生成
  - 尤其是包含大量短碎悬垂路径的模型

## 4. 复现步骤（修复前）
1. 导入包含大量短距离悬垂区域的模型。
2. 启用悬垂风扇相关功能并切片。
3. 打开生成的 G-code，搜索 `M106`。
4. 观察悬垂区域附近会出现高频 `M106` 切换。
5. 结果: 短时间内反复开关风扇，控制抖动明显。

## 5. 根因分析
- 原逻辑基于路径 marker（如 `;_OVERHANG_FAN_START/END`）即时切换风扇，请求粒度过细。
- 当悬垂路径被切分为大量短段时，会在相邻路径段之间频繁产生“开启/关闭”请求。
- 风扇控制是时间系统（机械惯性 + 风道惯性），按单段路径即时切换会造成典型抖动。
- 缺少以下约束:
  - 最短激活持续时间（短悬垂不该触发）
  - 最小保持时间（开启后不应立刻关闭）
  - 短间隙合并（碎片悬垂应视为一簇）
  - 最小风扇变化量（小幅变化无需发 `M106`）

## 6. 修复策略
- 在 `CoolingBuffer` 层引入“时间域风扇平滑/去抖”逻辑（不改几何路径生成层）。
- 将悬垂风扇 marker 视为“原始请求”，按 `CoolingLine::time()` 在层内顺序扫描后再决定是否真正输出 `M106`。
- 增加以下控制策略:
  - `min_activation_duration`: 连续悬垂时长不足阈值不升风扇
  - `min_hold_time`: 升风扇后至少保持一段时间
  - `merge_gap_time`: 短空档合并，不立即回落
  - `fan_min_delta`: 风扇变化幅度过小不发指令

## 7. 代码改动摘要
- 文件: `src/libslic3r/GCode/CoolingBuffer.cpp`
- 关键修改点:
  - 新增悬垂风扇时间域状态机 `OverhangFanPlannerState`
  - 使用 `line->time()` 进行按时间累计与决策（替代按单段 marker 即时切换）
  - 将 `TYPE_OVERHANG_FAN_START/END` 从“立即发风扇指令”改为“更新原始请求状态”
  - 新增 `sync_overhang_fan_plan(...)` 执行去抖与平滑
  - 新增 `emit_part_fan_if_needed(...)`，加入最小风扇变化量抑制
- 当前默认参数（代码内常量）:
  - `OVERHANG_FAN_MIN_ACTIVATION_DURATION = 0.7s`
  - `OVERHANG_FAN_MIN_HOLD_TIME = 1.2s`
  - `OVERHANG_FAN_MERGE_GAP_TIME = 0.20s`
  - `FAN_MIN_DELTA_PERCENT = 10%`

## 8. 验证清单
- [ ] 含大量短悬垂模型切片后，`M106` 数量明显减少。
- [ ] 短碎悬垂区不再出现高频 `M106` 抖动切换。
- [ ] 连续较长悬垂区仍能正确触发悬垂风扇。
- [ ] 风扇恢复到基础风扇值时不存在明显异常延迟（超出策略预期）。
- [ ] 普通非悬垂模型切片结果与风扇行为无回归异常。
- [ ] 支撑界面风扇逻辑（`SUPP_INTERFACE_FAN`）行为不受意外影响。

## 9. 风险与回滚
- 回滚方式: 回退 `src/libslic3r/GCode/CoolingBuffer.cpp` 中悬垂风扇时间域状态机相关修改。
- 风险等级: `中`（影响风扇调度行为，但改动点集中在 CoolingBuffer 输出阶段）。
- 重点关注副作用:
  - 极短 bridge/悬垂段可能因阈值过高而不触发风扇
  - 参数对不同材料（PLA/PETG/ABS）适配性差异
  - 与现有支撑界面风扇/恢复风扇逻辑的交互

## 10. 后续建议
- 将上述时间域参数改为可配置项（当前为代码常量）。
- 为 bridge 增加独立 marker 或 override 策略（允许桥接快速升风扇）。
- 增加自动化回归检查：统计特定模型切片后 `M106` 次数与分布。
