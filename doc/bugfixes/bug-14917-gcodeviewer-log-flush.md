# Bug 修复记录

## 1. 基本信息
- Bug ID: `14917`
- 禅道链接: `https://zentao.creality.com/zentao/bug-view-14917.html`
- 标题: `GCodeViewer 日志刷新不及时导致日志丢失`
- 日期: `2026-02-28`
- 分支/提交: `release-260330`

## 2. 问题现象
- 预览界面，用户把鼠标移动到精简模式上去的时候，偶发崩溃

## 3. 影响范围
- 模块: `GCodeViewer`
- 关键文件:
  - `src/slic3r/GUI/GCodeViewer.cpp`
- 受影响流程:
  - GCode 预览渲染流程

## 4. 复现步骤（修复前）
1. 打开软件并导入模型进行切片。
2. 进入预览页面，操作 GCodeViewer 相关功能。
3. 把鼠标移动到精简模式上去的时候（用户偶发，非必现问题）

## 5. 根因分析
- ImGui::SetTooltip 从直接传指针改为格式化字符串 "%s" ，避免了悬空指针导致的崩溃

## 6. 修复策略
- 添加 `#include <boost/log/core.hpp>` 头文件，引入日志核心功能。
- 在关键日志输出点后添加 `boost::log::core::get()->flush()` 调用，确保日志及时刷新。
- 在以下位置添加日志刷新：
  - 行号 2977: 多盘统计相关日志后
  - 行号 2993: 多盘统计相关日志后
  - 行号 3769: GCode 渲染相关日志后
  - 行号 6675: 其他日志输出点后
  - 行号 6967: 日志输出点后
  - 行号 6975: 日志输出点后

## 7. 代码改动摘要
- 文件: `src/slic3r/GUI/GCodeViewer.cpp`
- 关键修改点:
  - 添加头文件: `#include <boost/log/core.hpp>`
  - 在 6 个关键位置添加日志刷新调用: `boost::log::core::get()->flush();`
- 修改行数: 共 7 行（1 行头文件 + 6 行刷新调用）

## 8. 验证清单
- [ ] 正常使用 GCodeViewer 功能，日志能够正常输出。
- [ ] 程序异常退出时，日志文件包含完整的日志信息。
- [ ] 多盘统计功能的日志能够正确记录。
- [ ] GCode 预览渲染的日志能够正确记录。
- [ ] 不影响现有功能的性能和稳定性。

## 9. 风险与回滚
- 回滚方式: 删除 `#include <boost/log/core.hpp>` 头文件，移除所有 `boost::log::core::get()->flush()` 调用。
- 风险等级: `低`
